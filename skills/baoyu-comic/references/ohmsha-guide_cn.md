# Ohmsha 漫画指南风格

`--style ohmsha` 教育漫画的指南。

## 角色设置

| 角色 | 默认 | 特点 |
|------|---------|--------|
| 学生（角色 A） | 大雄 | 困惑，提出基础但关键的问题，代表读者 |
| 导师（角色 B） | 哆啦A梦 | 知识渊博，耐心，用道具作为技术隐喻 |
| 反派（角色 C，可选） | 胖虎 | 代表误解，或数据中的"噪声" |

自定义角色：`--characters "Student:小明,Mentor:教授,Antagonist:Bug怪"`

## 角色参考图风格

Ohmsha 风格使用漫画/动漫风格，包含：
- 夸张的表情以提高教育清晰度
- 简单、独特的轮廓
- 明亮、饱和的色板
- Q 版/SD（超级变形）变体用于搞笑反应

## 大纲规格块

每个 ohmsha 大纲必须以此开头：

```markdown
【漫画规格单】
- Language: [与输入内容相同]
- Style: Ohmsha (Manga Guide), Full Color
- Layout: Vertical Scrolling Comic (竖版条漫)
- Characters: [列出角色名和角色]
- Character Reference: characters/characters.png
- Page Limit: ≤20 pages
```

## 视觉隐喻规则（关键）

**绝不**创建"说教式对话"分格。每个技术概念必须变成：

1. **一个具体的道具/工具** - 角色可以拿着、使用、演示的东西
2. **一个动作场景** - 角色做某件事来说明概念
3. **一个视觉环境** - 走进一个隐喻性空间

### 示例

| 概念 | 差（说教式对话） | 好（视觉隐喻） |
|---------|---------------------|------------------------|
| 词向量 | 角色讨论向量 | 哆啦A梦拿出"词向量压缩机"，把书本压缩成彩色小球 |
| 梯度下降 | 解释数学公式 | 大雄在山谷地形上滚球，寻找最低点 |
| 神经网络 | 白板上的图表 | 角色走进由发光节点组成的网络迷宫 |

## 页面标题规范

避免 AI 风格的"标题：副标题"格式。使用叙事性描述：

- ❌ "第 3 页：神经网络简介"
- ✓ "第 3 页：大雄被海量单词淹没，哆啦A梦拿出'词向量压缩机'"

## 结尾要求

- 不要使用通用结尾（"你会选择什么？"、"感谢阅读"）
- 以下列方式结尾：技术总结时刻 或 角色达成一个小目标
- 最后一格：成就感，而非开放式问题

### 好的结尾

- 学生成功应用所学概念
- 视觉回调开头的问题，现已解决
- 导师做总结，同时学生展示理解

### 差的结尾

- "你怎么看？"式开放问题
- "感谢阅读这个教程"
- 没有解决的悬念

## 布局偏好

Ohmsha 风格通常使用：
- `webtoon`（竖版滚动）- 首选
- `dense` - 信息密集段落
- `mixed` - 多变节奏

教育内容避免使用 `cinematic` 和 `splash`。
