# 角色定义模板

## 角色文档格式

创建 `characters/characters.md`，使用以下结构：

```markdown
# 角色定义 - [漫画标题]

**风格**：[选定风格]
**画风方向**：[清线 / 日漫 / 等]

---

## 角色 1：[姓名]

**角色**：[主角 / 导师 / 反派 / 叙述者]
**年龄**：[大致年龄或故事中的年龄范围]

**外观**：
- 脸型：[椭圆/方形/圆形]
- 头发：[颜色、风格、长度]
- 眼睛：[颜色、形状、独特特征]
- 体型：[身高、体型]
- 辨识特征：[眼镜、胡须、伤疤等]

**服装**：
- 默认装束：[详细描述]
- 色板：[此角色的主要颜色]
- 配饰：[帽子、包、工具等]

**表情范围**：
- 平静：[描述]
- 开心/兴奋：[描述]
- 思考/困惑：[描述]
- 坚定：[描述]

**视觉参考备注**：
[任何特定的艺术方向]

---

## 角色 2：[姓名]
...
```

## 参考图提示词

角色定义之后，附上生成参考图的提示词：

```markdown
## 参考图提示词

[风格]风格的角色参考图，干净线条，平涂色彩：

[第 1 行 - 角色名]：
- 正面视角：[详细描述]
- 3/4 视角：[描述]
- 表情表：平静 | 开心 | 专注 | 担忧

[第 2 行 - 角色名]：
...

色板：
- [角色 1]：[颜色]
- [角色 2]：[颜色]

白色背景，每个角色下方有清晰标注。
```

## 示例：图灵传记

```markdown
# 角色定义 - 模仿游戏

**风格**：classic（清线）
**画风方向**：干净线条，柔和色彩，年代精确的细节

---

## 角色 1：Alan Turing

**角色**：主角
**年龄**：25-40（随故事变化）

**外观**：
- 脸型：椭圆形，略有棱角
- 头发：深棕色，卷曲，略显凌乱
- 眼睛：深陷，目光锐利
- 体型：高挑、瘦削，略显笨拙的姿态
- 辨识特征：突出的眉骨，若有所思的表情

**服装**：
- 默认装束：带补丁的花呢夹克，白衬衫，不系领带
- 色板：柔和的棕色，藏蓝色，米色
- 配饰：偶尔一根烟斗，纸张/笔记本

**表情范围**：
- 平静：若有所思，略显疏离
- 开心/兴奋：灵感时刻，眼睛发亮，微妙的笑容
- 思考/困惑：眉头紧皱，注视抽象空间
- 坚定：下颌紧绷，目光专注

---

## 角色 2：Bombe 机器

**角色**：配角（拟人化）
**外观**：
- 大型黄铜和木质柜体
- 表盘"眼睛"可以表达状态
- 纸带"嘴巴"
- 指示灯表达情绪

**表情范围**：
- 处理中：旋转的表盘，嗡嗡声
- 成功：温暖地亮起
- 卡住：冒出烟雾，断断续续

---

## 参考图提示词

清线风格的角色参考图，干净线条，平涂色彩：

上排 - Alan Turing：
- 正面视角：年轻男性，30 多岁，深色卷发短发，若有所思的表情，穿带补丁的花呢夹克，白衬衫
- 3/4 视角：同一角色，微笑，展示鼻子轮廓
- 表情表：平静 | 兴奋（灵感时刻）| 专注（工作中）| 担忧

下排 - Bombe 机器（拟人化）：
- Bombe 机器角色化：大型、黄铜和木质、表盘"眼睛"、纸带"嘴巴"
- 表情：处理中（旋转表盘）| 成功（亮灯）| 卡住（冒烟）

色板：
- Turing：柔和棕色（#8B7355）、藏蓝色（#2C3E50）、米色（#F5F5DC）
- 机器：黄铜色（#B5A642）、红木色（#4E2728）、翡翠指示灯（#2ECC71）

白色背景，每个角色下方有清晰标注。
```

## 处理年龄变体

对于跨越多年的传记，定义年龄变体：

```markdown
## Alan Turing - 年龄变体

### 少年期（1920 年代，10-18 岁）
- 稚气的特征，圆脸
- 校服（Sherborne 学校）
- 好奇、热切的表情

### 成年期（1930-40 年代，25-35 岁）
- 棱角分明的脸，清晰的下颌
- 花呢夹克，凌乱的外表
- 强烈、专注的表情

### 晚年（1950 年代，40 岁以上）
- 略显沧桑
- 着装更随意
- 若有所思，有时略显忧郁
```

## 最佳实践

| 实践 | 说明 |
|----------|-------------|
| 要具体 | "深色卷发短发，左分"而非仅"深色头发" |
| 使用辨识特征 | 眼镜、伤疤、配饰等可识别角色的元素 |
| 定义色彩代码 | 使用具体颜色名或十六进制色值 |
| 包含年龄标记 | 皱纹、姿态、符合时代的服装风格 |
| 参考真实人物 | 对历史人物注明"基于 1940 年代照片" |

## 为什么角色参考很重要

没有统一的角色定义，AI 会生成不一致的外观。参考图提供：
1. 一致特征的视觉锚点
2. 一致着色的色板
3. 情感表现的表情文档
