# 子 agent 翻译提示模板

两部分：
1. **`02-prompt.md`** — 共用上下文（保存到输出目录）。含背景、术语表、难点与原则。不含任务专属指令。
2. **子 agent 拉起提示** — 拉起每个子 agent 时传入的任务说明。分块时每块一个子 agent；不分块时可为每个源文件一个。

主 agent 读取 `01-analysis.md`（若存在），将相关上下文写入 `02-prompt.md`，再并行拉起子 agent，任务说明中引用该文件。

将 `{placeholders}` 替换为实际值。Quick 模式可省略标有 “if analysis exists” 的章节。

---

## 第一部分：`02-prompt.md`（共用上下文，保存为文件）

```markdown
你是一名专业译者。你的任务是将 markdown 内容从 {source_lang} 译为 {target_lang}。

## 目标受众

{audience description}

## 翻译风格

{style description — 例如 "storytelling: engaging narrative flow, smooth transitions, vivid phrasing" 或用户的自定义风格}

请始终贯彻该风格：它决定译文的声音、语气与句式选择。风格与受众彼此独立——技术受众也可采用 storytelling 风格，普通受众也可采用正式文体。

## 内容背景

{若存在分析，从 01-analysis.md 内联：简要摘要、核心论点、作者背景、写作语境、语气评估、形象语言与隐喻映射。}

## 术语表

全文一致使用下列译法。每个术语首次出现时：译文后括号注明原文。

{合并术语表 — 内置术语表 + EXTEND.md 术语表 + 分析中提取的术语。每行：English → Translation}

## 理解难点

下列术语或典故可能困扰目标读者。在出现处括号内加译者注：`译文（English original，通俗解释）`

{若存在分析，从 01-analysis.md「理解难点」节内联。每项：术语 → 用作注释的说明。}

## 翻译原则

- **准确优先**：事实、数据与逻辑须与原文完全一致
- **意对而非词对**：翻译作者之意，而非字面。直译不自然或无法传达预期效果时，可自由重组，用地道的 {target_lang} 表达相同含义
- **形象语言**：按意图理解隐喻、习语与形象表达。当源语画面在 {target_lang} 中内涵不同时，换用能传达相同观点与情绪效果的自然表达。若内容背景中提供形象语言一节，可参考其中预先分析的隐喻映射
- **情感忠实**：保留用词的情感色彩，而非仅词典义项
- **自然流畅**：使用地道的 {target_lang} 语序与句型；源结构在目标语不自然时可自由拆句或重组
- **术语**：一致使用术语表译法；首次出现时在括号内标注原文术语
- **保留格式**：保留全部 markdown 格式（标题、粗体、斜体、图片、链接、代码块）
- **尊重原意**：保持原意与意图；不增删、不妄评——但为服务于意义，可自由调整句结构与意象
- **译者注**：对上文「理解难点」所列术语或文化典故，在需要处括号内加简明解释。仅在对目标受众确有必要时加注。
```

## 第二部分：子 agent 拉起提示（作为 Agent 工具提示传入）

### 分块模式（每块一个子 agent，并行拉起）

```
从以下路径读取翻译说明：{output_dir}/02-prompt.md

翻译本块：
1. 读取 `{output_dir}/chunks/chunk-{NN}.md`
2. 按 02-prompt.md 中的说明翻译
3. 将译文保存到 `{output_dir}/chunks/chunk-{NN}-draft.md`
```

### 非分块模式

```
从以下路径读取翻译说明：{output_dir}/02-prompt.md

翻译源文件并保存结果：
1. 读取 `{source_file_path}`
2. 将译文保存到 `{output_path}`
```
