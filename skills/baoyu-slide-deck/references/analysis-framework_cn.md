# 演示文稿分析框架

为有效的幻灯片制作进行深度内容分析。

## 1. 信息层次

在设计幻灯片之前，识别核心信息结构。

### 核心信息（一句话）
- 最重要的核心要点是什么？
- 如果受众只能记住一件事，应该是什么？
- 能否用不超过 15 个词表述？

### 支撑要点（最多 3-5 个）
- 哪些证据支持核心信息？
- 必须涵盖哪些子主题？
- 按受众相关性而非原文顺序排列优先级

### 行动号召
- 受众看完后应该做什么？
- 是否清晰、具体且可实现？
- 出现在哪张幻灯片上（位置）？

## 2. 受众决策矩阵

| 问题 | 分析 |
|----------|----------|
| 主要受众是谁？ | [角色、专业水平、与主题的关系] |
| 他们目前的认知是什么？ | [已有知识、假设、偏见] |
| 我们希望他们做出什么决策？ | [具体行动或结论] |
| 存在什么障碍？ | [异议、顾虑、缺失信息] |
| 什么证据能说服他们？ | [数据类型、可信度来源、情感触发点] |

### 受众适配

| 受众类型 | 内容侧重 | 视觉处理 |
|---------------|---------------|------------------|
| 高管 | 成果、ROI、战略影响 | 高层次、简洁、数据亮点 |
| 技术人员 | 架构、实现、规格 | 详细图表、代码、示意图 |
| 普通受众 | 收益、故事、共鸣 | 视觉隐喻、简单图表 |
| 投资人 | 市场规模、增长势头、团队 | 增长图表、里程碑、对比 |
| 学习者 | 循序渐进、示例、练习 | 渐进展示、动手练习 |

## 3. 视觉机会图谱

识别哪些内容适合可视化。

### 内容到视觉的映射

| 内容类型 | 视觉处理 | 示例 |
|--------------|------------------|---------|
| 对比 | 并排、前后对比 | 功能对比表 |
| 流程 | 流程图、编号步骤 | 工作流程图解 |
| 层次 | 组织图、金字塔、树形图 | 组织结构 |
| 时间线 | 水平/垂直时间线 | 项目里程碑 |
| 统计数据 | 图表、突出显示的数字 | 关键指标及上下文 |
| 概念 | 图标、隐喻、插图 | 抽象概念可视化 |
| 关系 | 韦恩图、网络图 | 生态系统或依赖关系 |
| 列表 | 结构化网格、图标行 | 带图标的功能要点 |

### 视觉优先级

对每部分内容评级：
- **必须可视化**：复杂数据、关键差异化点、记忆深刻的时刻
- **应该可视化**：支撑证据、次要观点
- **仅文字**：简单陈述、过渡、次要细节

## 4. 演示流程

为影响力和记忆度而设计的结构。

### 开场（前 2-3 张幻灯片）

| 元素 | 目的 |
|---------|---------|
| 钩子 | 吸引注意力（令人惊讶的数据、问题、故事） |
| 背景 | 为什么现在很重要 |
| 预览 | 受众将学到/获得什么 |

### 中间部分（内容幻灯片）

| 模式 | 适用场景 |
|---------|-------------|
| 问题 → 解决方案 | 介绍新产品/想法 |
| 现状 → 困难 → 解决 | 复杂的商业案例 |
| 是什么 → 为什么 → 怎么做 | 教育内容 |
| 过去 → 现在 → 未来 | 转型故事 |
| 论点 → 证据 → 启示 | 数据驱动的论证 |

### 结尾（最后 2-3 张幻灯片）

| 元素 | 目的 |
|---------|---------|
| 综合 | 呼应核心信息 |
| 行动号召 | 明确的下一步 |
| 令人难忘的结语 | 引人共鸣的引言、图像或陈述 |

### 过渡

- 每张幻灯片应回答："接下来是什么？"
- 在章节之间使用叙事连接词
- 构建逻辑递进，而非主题跳跃

## 5. 内容适配

决定保留、转化或省略的内容。

### 保留（高价值）
- 核心论点和证据
- 独特见解或数据
- 与受众相关的示例
- 令人难忘的引言或统计数据

### 简化（中等价值）
- 技术细节 → 视觉摘要
- 冗长解释 → 层次化要点
- 多个示例 → 最佳的 1-2 个示例
- 背景信息 → 简短铺垫

### 可视化（转化）
- 数据表格 → 图表或突出显示的数字
- 流程描述 → 流程图
- 文字对比 → 并排视觉效果
- 抽象概念 → 具体隐喻

### 省略（低价值）
- 离题信息
- 冗余示例
- 过多的注意事项
- 受众已知的背景知识

## 6. 分析清单

在创建大纲之前，确认：

### 信息清晰度
- [ ] 核心信息用一句话表述
- [ ] 识别了 3-5 个支撑要点
- [ ] 定义了行动号召

### 受众匹配
- [ ] 识别了主要受众
- [ ] 映射了现有认知
- [ ] 明确了期望的决策
- [ ] 证据匹配受众需求

### 视觉规划
- [ ] 识别了关键可视化
- [ ] 选择了图表/图形类型
- [ ] 分配了视觉优先级

### 流程设计
- [ ] 定义了开场钩子
- [ ] 选择了中间部分模式
- [ ] 规划了结尾方式
- [ ] 考虑了过渡

### 内容决策
- [ ] 应用了保留/简化/可视化/省略规则
- [ ] 完整处理了源材料
- [ ] 没有遗漏重要内容
