# 内容与风格规则

幻灯片内容质量和风格一致性的指导方针。

## 内容规则

### 1. 尊重读者注意力
- 每张幻灯片应传达一个主要观点
- 删除冗余信息
- 优先考虑清晰度而非全面性

### 2. 数据可溯源
- 所有统计数据必须标注来源
- 在包含数据的幻灯片上直接引用来源
- 使用具体数字而非模糊表述

### 3. 自包含提示词
- 每个细节都必须包含在图像提示词中
- 不使用外部引用（如"像第 2 张幻灯片那样"）
- 明确包含所有颜色、布局和内容

### 4. 无占位符
- 每个元素必须完全指定
- 不使用"[此处插入数据]"或"待定"
- 所有文字内容在生成前必须最终确定

## 风格规则

### 1. 叙事式标题
标题讲述故事，而非标注内容。

| 差 | 好 |
|-----|------|
| "关键统计数据" | "使用量在 6 个月内翻倍" |
| "我们的解决方案" | "一个平台取代五个工具" |
| "优势" | "团队每周节省 10 小时" |

### 2. 避免 AI 套话
删除以下模式：
- "深入探讨"、"探索"、"旅程"
- "让我们来看看"、"让我给你展示"
- "令人兴奋的"、"惊人的"、"革命性的"
- "总结一下"、"综上所述"

### 3. 有意义的封底
不仅仅是"谢谢"或"有问题吗？"

包含以下之一：
- 明确的行动号召
- 令人难忘的关键要点
- 发人深省的结束语
- 附带目的的联系信息

### 4. 一致的视觉语言
整个幻灯片保持：
- 相同的图标风格
- 相同的颜色使用模式
- 相同的布局网格系统
- 相同的字体层次

## 幻灯片结构

| 位置 | 类型 | 目的 |
|----------|------|---------|
| 1 | 封面 | 标题、视觉钩子、主题介绍 |
| 2 到 N-1 | 内容 | 关键要点、数据、解释 |
| N | 封底 | 总结、行动号召、令人难忘的结语 |

## 关键规格

| 规格 | 值 |
|---------------|-------|
| 宽高比 | 16:9（横向） |
| 幻灯片数量 | 根据内容动态调整 |
| 必需幻灯片 | 至少包含封面 + 封底 |
| 页脚 | 无（无幻灯片编号、Logo） |
| 语言优先级 | `--lang` → 源语言 → 询问用户 |
| 语气 | 直接、自信（避免 AI 短语） |

## 风格快速参考

| 风格 | 视觉概述 |
|-------|----------------|
| `sketch-notes` | 手绘、暖色调米白底、概念图标 |
| `blueprint` | 技术示意图、网格纹理、蓝色调 |
| `bold-editorial` | 高对比度、深色背景、杂志冲击力 |
| `vector-illustration` | 扁平矢量、黑色轮廓、复古色彩 |
| `minimal` | 最大留白、单一强调色、禅意 |
| `storytelling` | 全出血图像、电影感、情感化 |
| `warm` | 柔和渐变、圆润形状、健康配色 |
| `notion` | 仪表盘美学、干净的数据可视化、SaaS 风格 |
| `corporate` | 藏青/金色、结构化布局、商务感 |
| `playful` | 鲜艳的珊瑚色/青色/黄色、动感、活力 |

完整风格规格：`references/styles/<style>.md`
