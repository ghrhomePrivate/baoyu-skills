# 文字维度

控制封面图上的文字密度和信息层次。

## 可选值

| 值 | 标题 | 副标题 | 标签 | 视觉区域 |
|-------|:-----:|:--------:|:----:|:-----------:|
| `none` | - | - | - | 100% |
| `title-only` | ✓ | - | - | 85% |
| `title-subtitle` | ✓ | ✓ | - | 75% |
| `text-rich` | ✓ | ✓ | ✓ (2-4) | 60% |

## 详细说明

### none

纯视觉封面，无文字元素。

**使用场景**：
- 以摄影为主的封面
- 抽象艺术作品
- 纯视觉社交分享
- 标题从外部添加时

**构图**：
- 完整的视觉区域可用
- 无预留文字区域
- 重点在于视觉隐喻

### title-only

单个标题，最大冲击力。

**使用场景**：
- 大多数文章封面（默认）
- 清晰的单一信息
- 强品牌辨识

**构图**：
- 标题：突出放置
- 预留区域：顶部或底部 15%
- 视觉支撑标题信息

**标题准则**：
- 使用源内容中的原始标题或用户提供的标题
- 不要自行编造或修改标题
- 匹配内容语言

### title-subtitle

标题配辅助上下文。

**使用场景**：
- 需要补充说明的技术文章
- 带有集数/部分信息的系列
- 具有双重信息的内容

**构图**：
- 标题：主要元素
- 副标题：次要元素
- 预留区域：25%
- 标题/副标题间的清晰层次

**标题准则**：
- 使用源内容中的原始标题或用户提供的标题
- 不要自行编造或修改标题

**副标题准则**：
- 补充或提供标题上下文
- 可包含系列名称、作者、日期
- 比标题小、不那么突出

### text-rich

信息密集的封面，包含多个文字元素。

**使用场景**：
- 信息图风格的封面
- 带有详情的活动公告
- 带有功能特性的推广材料
- 包含多个要点的内容

**构图**：
- 标题：主要焦点
- 副标题：辅助信息
- 标签：2-4 个关键词标签
- 预留区域：40%
- 清晰的视觉层次

**标题准则**：
- 使用源内容中的原始标题或用户提供的标题
- 不要自行编造或修改标题

**标签准则**：
- 最多 2-4 个标签
- 短关键词（每个 1-2 个词）
- 以徽章/标签形式定位
- 可突出：分类、日期、作者、主要特性

## 类型兼容性

| 类型 | none | title-only | title-subtitle | text-rich |
|------|:----:|:----------:|:--------------:|:---------:|
| hero | ✓ | ✓✓ | ✓✓ | ✓ |
| conceptual | ✓✓ | ✓✓ | ✓ | ✓ |
| typography | ✗ | ✓ | ✓✓ | ✓✓ |
| metaphor | ✓✓ | ✓ | ✓ | ✗ |
| scene | ✓✓ | ✓ | ✓ | ✗ |
| minimal | ✓✓ | ✓✓ | ✓ | ✗ |

✓✓ = 强烈推荐 | ✓ = 兼容 | ✗ = 不推荐

## 自动选择

当 `--text` 未指定时，根据信号选择：

| 信号 | 文字级别 |
|---------|------------|
| 纯视觉、摄影、抽象、艺术 | `none` |
| 文章、博客、标准封面 | `title-only` |
| 系列、教程、带上下文的技术内容 | `title-subtitle` |
| 公告、功能特性、多要点、信息图 | `text-rich` |

默认：`title-only`
