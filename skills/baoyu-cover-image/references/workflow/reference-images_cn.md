# 参考图片处理

封面生成中用户提供参考图片的处理指南。

## 输入检测

| 输入类型 | 操作 |
|----------|------|
| 提供了图片文件路径 | 复制到 `refs/` → 可使用 `--ref` |
| 对话中有图片（无路径） | **用 AskUserQuestion 请求用户提供文件路径** |
| 用户无法提供路径 | 口头提取风格/配色 → 附加到提示词（不添加 frontmatter 引用） |

**关键**：仅当文件已**实际保存**到 `refs/` 目录时，才在提示词的 frontmatter 中添加 `references`。

## 文件保存

**如果用户提供了文件路径**：
1. 复制到 `refs/ref-NN-{slug}.{ext}`（NN = 01, 02, ...）
2. 创建描述文件：`refs/ref-NN-{slug}.md`
3. 在继续之前验证文件存在

**描述文件格式**：
```yaml
---
ref_id: NN
filename: ref-NN-{slug}.{ext}
usage: direct | style | palette
---
[用户的描述或自动生成的描述]
```

| 用途 | 使用场景 |
|------|----------|
| `direct` | 模型直接查看参考图；输出中必须出现人物时必选 |
| `style` | 仅提取视觉风格（不用于必须出现的人物） |
| `palette` | 仅提取配色方案 |

## 口头提取（无文件）

当用户无法提供文件路径时：
1. 视觉分析图片，提取：颜色、风格、构图
2. 创建 `refs/extracted-style.md` 保存提取信息
3. 不要在提示词 frontmatter 中添加 `references`
4. 将提取的风格/颜色直接附加到提示词文本

## 深度分析 ⚠️ 关键

参考图是高优先级输入。提取**具体、明确、可复现**的元素：

| 分析 | 描述 | 示例（好 vs 差） |
|------|------|------------------|
| **品牌元素** | Logo、文字标识、特定排版 | 好："Logo 使用竖直平行线构成 'm'" / 差："有个 logo" |
| **标志性图案** | 独特装饰纹样、质感 | 好："交织曲线形成菱形网格" / 差："有图案" |
| **色彩方案** | 关键颜色的精确 hex 值 | 好："#2D4A3E 深青、#F5F0E0 奶油色" / 差："深浅色搭配" |
| **布局结构** | 具体的空间排列 | 好："底部 30% 为深色品牌横幅" / 差："有个横幅" |
| **字体排版** | 字体风格、粗细、间距、大小写 | 好："全大写，宽字间距" / 差："有文字" |
| **内容/主题** | 参考图所描绘的内容 | 事实性描述 |
| **使用建议** | `direct` / `style` / `palette` | 基于分析结果 |

**输出格式**：以列表项列出每个元素，可直接复制粘贴到提示词中作为强制指令。

### 人物分析 ⚠️ 如果参考图包含人物

使用 `usage: direct` 让模型直接查看参考图。此外，对每个人物描述：**外貌**、**姿势**、**服装** → 附带**转化规则**（按渲染风格进行风格化）。

| 提取 | 好 | 差 |
|------|----|----|
| 外貌 | "女性：长波浪金发，亲切微笑" | "一个女人" |
| 姿势 | "站立，面向镜头，自信姿态" | "站着" |
| 服装 | "深色T恤，商务休闲" | "正式" |
| 转化 | "扁平矢量卡通风，保留发色和服装" | "做成卡通" |

使用 `usage: direct`。将每个人物作为 MUST/REQUIRED 提示词指令输出。

## 验证输出

**已保存文件**：
```
参考图片已保存：
- ref-01-{slug}.png ✓（可使用 --ref）
- ref-02-{slug}.png ✓（可使用 --ref）
```

**已提取风格**：
```
已提取参考风格（无文件）：
- 颜色：#E8756D 珊瑚色、#7ECFC0 薄荷色...
- 风格：极简扁平矢量、干净线条...
→ 将附加到提示词文本（非 --ref）
```

## 优先级规则

当用户提供参考图时，它们是**高优先级**的：

- **参考图覆盖默认值**：如果参考图与首选配色/渲染冲突，参考图优先
- **具体 > 抽象**：提取具体元素 — 而非模糊的"简洁风格"
- **强制性语言**：在提示词中对参考图元素使用"MUST"、"REQUIRED"
- **输出中可见**：生成后验证元素是否存在；如果缺失则加强提示词
