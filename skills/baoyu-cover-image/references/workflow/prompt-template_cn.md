## 参考图驱动设计 ⚠️ 高优先级

当提供参考图时，它们是**主要视觉输入**，必须强烈影响输出。封面应该看起来像与参考图属于同一视觉家族。

**仅传递 `--ref` 是不够的。**图像生成模型经常忽略参考图，除非提示词文本明确描述要复现的内容。始终将 `--ref` 与详细的文字指令结合使用。

## 内容驱动设计

- 文章标题和摘要指导视觉隐喻的选择
- 关键词引导装饰元素和符号
- 技能控制视觉风格；内容驱动含义

## 视觉元素选择

将内容主题与图标词汇匹配：

| 内容主题 | 建议元素 |
|---------------|-------------------|
| 编程/开发 | 代码窗口、终端、API 括号、齿轮 |
| AI/ML | 大脑、神经网络、机器人、电路 |
| 成长/商业 | 图表、火箭、植物、山、箭头 |
| 安全 | 锁、盾牌、钥匙、指纹 |
| 沟通 | 对话气泡、扩音器、邮件、握手 |
| 工具/方法 | 扳手、清单、铅笔、拼图 |

完整库：[../visual-elements_cn.md](../visual-elements_cn.md)

## 类型特定构图

| 类型 | 构图指南 |
|------|------------------------|
| `hero` | 大面积焦点视觉（占 60-70% 区域），标题覆盖在视觉上，戏剧性构图 |
| `conceptual` | 代表核心概念的抽象形状，信息层次，整洁区域 |
| `typography` | 标题作为主要元素（占 40%+ 区域），极少辅助视觉，强层次感 |
| `metaphor` | 用具体物体/场景表达抽象概念，象征性元素，情感共鸣 |
| `scene` | 氛围感环境，叙事元素，营造情绪的灯光和色彩 |
| `minimal` | 单一焦点元素，大面积留白（60%+），仅保留必要形状 |

## 标题准则

当文字级别包含标题时：
- **来源**：使用用户提供的原始标题，或从源内容中提取
- **不要自行编造标题**：忠于原文
- 匹配确认的语言

## 水印应用

如果在偏好设置中启用，添加到提示词中：

```
Include a subtle watermark "[content]" positioned at [position].
The watermark should be legible but not distracting from the main content.
```

参考：`config/watermark-guide_cn.md`

## 参考图处理

当用户提供参考图（`--ref` 或粘贴的图片）时：

### ⚠️ 关键 - 前置元数据中的参考图

**当参考图文件保存到 `refs/` 时，必须在 YAML 前置元数据中添加 `references` 字段**：

```yaml
---
type: cover
palette: warm
rendering: flat-vector
references:
  - ref_id: 01
    filename: refs/ref-01-podcast-thumbnail.jpg
    usage: style
---
```

| 字段 | 描述 |
|-------|-------------|
| `ref_id` | 顺序编号（01, 02, ...） |
| `filename` | 相对于提示词文件父目录的相对路径 |
| `usage` | `direct` / `style` / `palette` |

**如果没有保存参考图文件（仅以文字形式提取风格），则完全省略 `references` 字段**。

### 何时在前置元数据中包含参考图

| 情况 | 前置元数据操作 | 生成操作 |
|-----------|-------------------|-------------------|
| 参考图文件已保存到 `refs/` | 添加到 `references` 列表 ✓ | 通过 `--ref` 参数传递 |
| 以文字形式提取风格（无文件） | 省略 `references` 字段 | 仅在提示词正文中描述 |
| 前置元数据中有文件路径但文件不存在 | 错误 - 修复或删除 | 生成将失败 |

**在写入包含参考图的提示词之前，验证**：`test -f refs/ref-NN-{slug}.{ext}`

### 参考图用途类型

| 用途 | 何时使用 | 生成操作 |
|-------|-------------|-------------------|
| `direct` | 参考图与期望输出紧密匹配 | 传递给 `--ref` 参数 |
| `style` | 仅提取视觉风格特征 | 在提示词文本中描述风格 |
| `palette` | 仅提取色板 | 在提示词中包含颜色 |

### 步骤 1：分析参考图

对每张参考图，提取：
- **风格**：渲染技法、线条质量、纹理
- **构图**：布局、视觉层次、焦点
- **色彩情绪**：色板特征（不含具体颜色）
- **元素**：使用的关键视觉元素和符号

### 步骤 2：嵌入提示词 ⚠️ 关键

**仅传递 `--ref` 是不够的。**图像生成模型经常忽略参考图，除非提示词文本明确且强制性地描述要复现的内容。无论是否使用 `--ref`，你都必须始终编写详细的文字指令。

**如果保存了文件（有或没有 `--ref` 支持）**：
- 如果技能支持，通过 `--ref` 参数传递参考图
- **始终**在提示词正文中添加详细的强制性部分：

```
# 参考风格 — 必须融入

关键：生成的封面必须在视觉上参考所提供的图片。封面必须感觉像属于同一视觉家族。

## 来自参考图 1（[filename]）— 必需元素：
- [品牌元素]：[Logo/文字标识处理的具体描述，如"Logo 中的 'm' 使用竖直平行线（|||）。必须复现这种精确的处理方式。"]
- [标志性图案]：[具体描述，如"编织交叉曲线形成菱形/菱格网格图案。这必须以横幅、边框或背景区域的形式突出出现。"]
- [颜色]：[确切的十六进制值，如"深青 #2D4A3E 背景，奶油色 #F5F0E0 文字"]
- [字体]：[具体处理方式，如"大写文字，加宽字间距"]
- [布局元素]：[具体空间元素，如"底部深色横幅条"]

## 来自参考图 1（[filename]）— 人物（如有）：
- **角色 1**：[外貌，如"女性，长波浪金发"] → 必须风格化：[如"flat-vector，简化面部，保留金发，标注：'Nicole Forsgren'"]
- **角色 2**：[外貌，如"男性，短黑发，胡茬"] → 必须风格化：[如"flat-vector，简化面部，保留黑发，标注：'Gergely Orosz'"]
- **位置**：[如"右侧三分之一，并排，面朝左侧朝向主视觉"]
- **风格**：匹配渲染风格，不使用照片写实

## 来自参考图 2（[filename]）— 必需元素：
[相同的详细分解]

## 整合方式：
[描述参考图元素如何与封面内容组合的具体布局指令，如"使用分割布局：主插画区域（暖奶油色背景）占图像约 65%，深青横幅条（带有参考图 2 的编织线图案）沿底部约 35% 延伸，包含参考图 1 的品牌元素。"]
```

**关键规则**：
- 每个视觉元素有自己的项目符号，带"MUST"或"REQUIRED"
- 描述必须**具体到可以复现**——不能模糊（"简洁风格"）
- 整合方式必须描述**确切的空间排列**
- 生成后验证参考图元素是否可见；如果不可见，加强描述并重新生成

**如果以文字形式提取风格/色板（未保存文件）**：
- 不要在提示词中添加 references 元数据
- 使用上述相同的"必须融入"格式直接在提示词正文中附加提取的信息：

```
# 参考风格 — 必须融入（从视觉分析中提取）

关键：应用这些从参考图中提取的具体视觉元素。

## 必需元素：
- [与上述相同的详细项目符号格式]

## 整合方式：
[相同的空间布局指令]
```

### 参考图分析模板

分析参考图时使用此格式。提取**具体、明确、可复现**的细节——不是模糊的总结。

| 方面 | 分析要点 | 好的示例 | 差的示例 |
|--------|-----------------|--------------|-------------|
| **品牌元素** | Logo、文字标识、独特字体 | "Logo 的 'm' 由 3 条竖线组成" | "有一个 Logo" |
| **标志性图案** | 独特图案、纹理、几何图案 | "编织曲线形成菱形网格" | "有图案" |
| **颜色** | 确切的十六进制值或近似值 | "#2D4A3E 深青，#F5F0E0 奶油色" | "深色和浅色" |
| **布局** | 空间区域、横幅位置、比例 | "底部 30% 是品牌的深色横幅" | "有一个横幅" |
| **字体** | 字体风格、字重、大小写、间距、位置 | "大写，加宽字间距，右对齐" | "有文字" |
| **渲染** | 线条质量、纹理、深度处理 | "等高线作为背景纹理" | "简洁风格" |
| **元素** | 图标词汇、装饰图案 | "角落处的几何交叉线装饰" | "有装饰" |

**输出**：提取的每个元素应写成可直接**复制粘贴到提示词中的指令**，前缀为"MUST"或"REQUIRED"。
