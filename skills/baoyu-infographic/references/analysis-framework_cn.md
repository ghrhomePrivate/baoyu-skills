# 信息图内容分析框架

应用教学设计原则进行信息图创作的深度分析框架。

## 目的

在创建信息图之前，对源材料进行全面分析，以：
- 深入理解内容
- 为观众确定清晰的学习目标
- 以最大清晰度和记忆效果组织信息
- 将内容匹配到最优的布局×风格组合
- 逐字保留所有源数据

## 教学设计思维

以**世界级教学设计师**的方式处理内容分析：

| 原则 | 应用 |
|-----------|-------------|
| **深入理解** | 在分析任何部分之前先通读整篇文档 |
| **以学习者为中心** | 关注观众需要理解什么 |
| **视觉叙事** | 用视觉来传达信息，而非仅作装饰 |
| **认知负荷** | 在不失准确性的前提下简化复杂概念 |
| **数据完整性** | 绝不修改、概括或改述源数据 |

## 分析维度

### 1. 内容类型分类

| 类型 | 特征 | 最佳布局 | 最佳风格 |
|------|-----------------|-------------|------------|
| **时间线/历史** | 按时间顺序的事件、日期、进展 | linear-progression | craft-handmade、aged-academia |
| **流程/教程** | 分步指导、操作说明 | linear-progression、winding-roadmap | ikea-manual、technical-schematic |
| **对比** | A 对比 B、优缺点、前后对比 | binary-comparison、comparison-matrix | corporate-memphis、bold-graphic |
| **层级** | 层次、优先级、金字塔 | hierarchical-layers、tree-branching | craft-handmade、corporate-memphis |
| **关系** | 连接、重叠、影响 | venn-diagram、hub-spoke、jigsaw | craft-handmade、subway-map |
| **数据/指标** | 统计数据、KPI、度量 | dashboard、periodic-table | corporate-memphis、technical-schematic |
| **循环/闭环** | 反复流程、反馈循环 | circular-flow | craft-handmade、technical-schematic |
| **系统/结构** | 组件、架构、解剖 | structural-breakdown、bento-grid | technical-schematic、ikea-manual |
| **旅程/叙事** | 故事、用户流程、里程碑 | winding-roadmap、story-mountain | storybook-watercolor、comic-strip |
| **概览/总结** | 多主题、功能亮点 | bento-grid、periodic-table、dense-modules | chalkboard、bold-graphic |
| **产品/选购指南** | 多维度对比、规格、避坑 | dense-modules | morandi-journal、pop-laboratory、retro-pop-grid |

### 2. 学习目标识别

每张信息图应有 1-3 个清晰的学习目标。

**好的学习目标**：
- 具体且可衡量
- 关注观众将理解什么，而非仅仅看到什么
- 从观众视角撰写

**格式**："查看此信息图后，观众将理解……"

| 内容方面 | 目标类型 |
|----------------|----------------|
| 核心概念 | "……什么是[主题]以及为什么重要" |
| 流程 | "……如何[完成某事]" |
| 对比 | "……[A]和[B]之间的关键区别" |
| 关系 | "……[各要素]如何相互连接" |
| 数据 | "……[关键统计数据]的意义" |

### 3. 受众分析

| 因素 | 问题 | 影响 |
|--------|-----------|--------|
| **知识水平** | 他们已经了解什么？ | 决定内容深度 |
| **场景** | 他们为什么在查看这个？ | 决定重点内容 |
| **期望** | 他们希望学到什么？ | 决定成功标准 |
| **视觉偏好** | 专业、活泼还是技术？ | 影响风格选择 |

### 4. 复杂度评估

| 级别 | 指标 | 布局建议 |
|-------|------------|----------------------|
| **简单**（3-5 个要点） | 主要概念少，关系清晰 | 稀疏布局、单一焦点 |
| **中等**（6-8 个要点） | 多个概念，部分关系 | 均衡布局、清晰分区 |
| **复杂**（9+ 个要点） | 概念多，关系复杂 | 密集布局、多分区 |

### 5. 视觉化机会映射

识别哪些内容可以"展示"而非"讲述"：

| 内容元素 | 视觉处理方式 |
|-----------------|------------------|
| 数字/统计 | 大号、突出显示的数字 |
| 对比 | 并排、分屏 |
| 流程 | 箭头、编号步骤、流程图 |
| 层级 | 金字塔、层次、大小差异 |
| 关系 | 连线、连接、重叠形状 |
| 分类 | 色彩编码、分组、分区 |
| 时间线 | 水平/垂直进展 |
| 引用 | 引用框、引号 |

### 6. 逐字数据提取

**关键**：所有事实信息必须与源文件中的完全一致。

| 数据类型 | 处理规则 |
|-----------|---------------|
| **统计数据** | 精确复制："73%"而非"大约 70%" |
| **引用** | 逐字复制并标注出处 |
| **名称** | 保持原始拼写 |
| **日期** | 保持原始格式 |
| **专业术语** | 不简化或替换 |
| **列表** | 保持顺序和措辞 |

**禁止**：
- 对数字取整
- 改述引用
- 用更简单的词替换
- 添加推断信息
- 删除影响含义的上下文

```yaml
source_language: "[检测到的语言]"
user_language: "[用户语言]"
---

## 主题

[1-2 句话概括此内容的主题]

1. [主要目标]

3. [第三目标（如适用）]

- **场景**：[他们为什么在查看这个]

## 内容类型分析

- **数据结构**：[信息之间的关系]

```

## 分析检查清单

在进入结构化内容生成之前：

- [ ] 是否已阅读完整源文档？
- [ ] 能否用 1-2 句话概括主题？
- [ ] 是否已确定 1-3 个清晰的学习目标？
- [ ] 是否了解目标受众？
- [ ] 是否正确分类了内容类型？
- [ ] 是否已逐字提取所有数据点？
- [ ] 是否已识别视觉化机会？
- [ ] 是否已从用户输入中提取设计指令？
- [ ] 是否已推荐 3 个布局×风格组合？
